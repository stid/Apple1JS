Key meanings:

- **artifact-root** — where LCD's durable artifacts live (`MAP.md`, `DECISIONS.md`, and the
  `work/<slug>/` per-work-item folders). Chosen once at onboarding; everything else derives from it.
- **map / decisions / work-item-dir** — resolved paths the suite reads; derived from `artifact-root`.
- **test-placement** — where the Deep lane's `/stid-lcd:test-gen` must write tests so Vitest
  actually collects them. The default `tests/<slug>/<surface>` is **not** matched by this project's
  discovery glob and would silently never run. `<area>` is the source folder under test (e.g.
  `core`, `apple1`, `components`, `services`); `<Name>` the subject; `<slug>`/`<surface>` per the
  LCD work-item slug and surface token.
- **test-discovery-glob** — Vitest's `include` from `vitest.config.ts`; the reason placement
  matters. `globals: true` is set, so test files need no `vitest` import (but it's harmless).
- **scoped-test / bail / single-test** — red-green loop commands; `{path}` and `{name}` are
  substituted. These use `vitest run` (NOT the bare `test` script, which is `vitest --silent` =
  watch-ish and the `test:watch` script is true watch mode — either would hang the autonomous loop).
- **gate** — `yarn test:ci` runs `pretest` (`lint` + `type-check`) then `vitest run` (the `T-final` task).
- **eval** — `n/a` here: a cycle-accurate emulator is deterministic with no scored/LLM output. The
  JS engine acts as a reference oracle for WASM parity, but that's a deterministic equality check,
  not an `EVAL` quality threshold.
- **maintenance-bundle** — what `/stid-lcd:tidy` runs: the full gate, a dependency audit
  (`yarn outdated`), and a markdown lint pass.

## Constitution notes

> Must-restate constraints from this project's `CLAUDE.md` + `.claude/rules/*` + `MAP.md`
> invariants that a Deep-lane `plan.md` Constitution check and the red-green loop should honor.
> Repeated here so they survive on a fresh clone without the maintainer's machine.

- **Never commit to `master`** — always a feature branch (`feat/ fix/ refactor/ docs/ test/ chore/`).
- **Dual-engine parity** — JS and WASM 6502 engines must stay feature-equal; test **both** for every
  engine change. The JS engine is the reference oracle / always-available fallback.
- **No `any` types** (strict mode); **no `console.log`** (use `LoggingService`); **no hardcoded
  colors** (design tokens — except the CRT display).
- **Bump `src/version.ts`** before every PR — single source of truth (patch/minor/major per branch prefix).
- **Stateful/inspectable contracts** (`IVersionedStatefulComponent`, `IInspectableComponent`) are
  scoped to the **core/IO layer**, not React presentational components.
- **Use the `Formatters` utility** (e.g. `Formatters.hexWord(addr)`) — never manual `toString(16)`;
  worker comms go through typed `sendWorkerMessage(...)`.
- **WASM build is speed-first** (`opt-level = 3`, ~155 KB); don't restore the size-first (<100 KB)
  target without approval.
